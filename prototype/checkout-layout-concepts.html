Checkout alignment & UX concepts

Live checkout had a DOM close-tag bug plus a shrink-to-fit centered shell, so step 3 looked left-shifted. Below: two layout concepts that keep the frame stable while the user moves through details → payment → confirmation.

Concept A — Fixed two-column stage frame

One permanent grid: form column left, summary column right. Steps swap inside the left column only. Shell width never changes, so summary and fields stay aligned across steps.

  • Best for desktop booking. Cart + customer + payment status stay visible while paying.
  • No horizontal jump. Grid tracks are fixed; only panel content swaps.
  • Trade-off: On narrow screens summary stacks above the form (order: summary → step).

Concept B — Single centered column + sticky progress

One narrow, always-centered column. A sticky step bar anchors orientation; a compact order strip sits above the form. Ideal when you want focus and identical alignment on phone and desktop.

  • Best for mobile-first. No two-column reflow surprises.
  • Seamless step feel. Progress is always in the same place; content swaps below.
  • Trade-off: Less room for a rich live summary beside payment (summary becomes a strip).
